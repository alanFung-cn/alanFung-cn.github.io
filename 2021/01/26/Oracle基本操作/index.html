创建用户、表空间、分配权限

SQL> create temporary tablespace van_temp
  2  tempfile 'C:\app\Alan\product\oracleData\ctp\van_temp.dbf'
  3  size 50m
  4  autoextend on
  5  next 50m maxsize 20480m
  6  extent management local;

表空间已创建。

SQL> create tablespace van_data
  2  logging
  3  datafile 'C:\app\Alan\product\oracleData\ctp\van_data.dbf'
  4  size 50m
  5  autoextend on
  6  next 50m maxsize 20480m
  7  extent management local;

表空间已创建。

SQL> create user van identified by admin123
  2  default tablespace van_data
  3  temporary tablespace van_temp;

用户已创建。

SQL> grant connect,resource,dba to van;

授权成功。

//查询临时文件存放位置
select name from v$tempfile;
//查询数据文件存放位置
select name from v$datafile;

//修改用户表空间
alter user van default tablespace van_data;
//修改用户临时表空间
alter user van temporary tablespace van_temp;

创建Directory

1、新建directory的语法

CREATE [OR REPLACE] DIRECTORY directory AS 'pathname';

例如： 

create or replace directory dump_dir as 'D:\dump\dir';

这样把目录d:\dump\dir设置成dump_dir代表的directory

2、查询当前已有的directory 

select * from dba_directories;

3、给制定用户赋权

grant read,write on directory dump_dir to user01

--dumo_dir=>已有的directory
--user01=>要赋权的用户

4、删除 

drop directory dump_dir;

导入dmp文件

impdp username/password@ORCL DIRECTORY=dir DUMPFILE=testFile.dmp REMAP_TABLESPACE=old_tableSpace:new_tableSpace REMAP_SCHEMA=old_user:new_user

--username=>用户名
--password=>密码
--ORCL=>主机名
--dir=>Directory名
--testFile=>文件名，需要放在Directory指向的目录下
--REMAP_TABLESPACE=>原有tableSpace名称：新tableSpace名称
--REMAP_SCHEMA=>原有用户名：新用户名

impdp van/admin123@orcl DIRECTORY=ctpBase_dir dumpfile=testData.dmp

impdp ctpBase/admin123@orcl DIRECTORY=ctpBase_dir dumpfile=testData.dmp REMAP_TABLESPACE=TBS_REPORT:ctpBase_data REMAP_SCHEMA=VAN:ctpBase

几种错误原因

impdp命令使用错误

(图片来源：https://www.cnblogs.com/zuiyue_jing/p/10635045.html)

出现这种错误表示要导入的文件路径不正确，不能直接把文件的绝对路径写入，应该使用Directory的方式来导入

目录没有赋权限

(图片来源：https://www.cnblogs.com/zuiyue_jing/p/10631906.html)

连接到: Oracle Database 11g Enterprise Edition Release 11.2.0.4.0 -64bit Production
With the Partitioning, OLAP, Data Mining and Real Application Testing options
ORA-39002: 操作无效
ORA-39070: 无法打开日志文件。
ORA-29283: 文件操作无效
ORA-06512: 在 "SYS.UTL_FILE", line 536
ORA-29283: 文件操作无效等类似的错误。

提示类似这种类型的错误表示当前用户没有访问指定DIRECTORY的权限，需要进入Sys用户给这个用户赋权，赋权语句：

GRANT read,write ON DIRECTORY dir_dump TO public;
